# 接口路径规范汇总

> 版本：v1.0  
> 目标：统一路径风格，消除 `/api/v1/`、`/api/user/`、`/api/` 混用问题，为前后端联调提供单一参考。

---

## 一、路径命名约定

| 规则 | 说明 | 示例 |
|------|------|------|
| 统一前缀 | 所有接口使用 `/api/` | ~~`/api/v1/`~~ → `/api/` |
| 资源复数 | 集合资源用复数名词 | `/api/rides`、`/api/vehicles` |
| 动词禁止出现在路径中 | 用 HTTP 方法表达操作 | ~~`/api/rides/publish`~~ → `POST /api/rides` |
| 子资源用斜杠 | 从属关系用路径嵌套 | `/api/vehicles/:id/default` |
| 操作例外 | 非 CRUD 动作可用子路径动词 | `/api/auth/logout`、`/api/sms/send` |
| 连字符分隔 | 多词路径用 `-` 不用 `_` | `/api/find-ride` ~~`/find_ride`~~ |
