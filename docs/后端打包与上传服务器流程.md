# 后端打包 dist 与上传服务器流程

本文档说明如何将后端打包为 `dist` 并上传到公网服务器部署。

**说明**：测试环境服务器在腾讯云，**由陈柯负责上传与部署**。小组其他成员可按下方流程在本地打包或了解整体步骤；需要更新测试环境时，将打包好的 dist 或相关说明交给陈柯，由陈柯上传到腾讯云服务器并完成部署。

---

## 一、本地打包

在项目 **backend** 目录下执行：

```bash
npm run build
```

会在 backend 下生成 **dist/** 文件夹，包含：

- `src/`（全部源码）
- `package.json`
- `package-lock.json`（若存在）

**不包含**：`node_modules`、`.env`（依赖在服务器上安装，环境变量在服务器上配置）。

---

## 二、上传到腾讯云服务器（由陈柯操作）

在本地得到 **dist** 后，可直接查看服务器上的目标部署目录（如 `/var/www/carpooling-api`），将本机 `backend/dist` 中的全部内容通过文件传输（拖拽或复制）上传到该目录即可。

---

## 三、服务器上的操作

1. **进入部署目录**

   ```bash
   cd /var/www/carpooling-api
   ```

2. **安装生产依赖**

   ```bash
   npm install --production
   ```

3. **配置环境变量**（可选）  
   新建 `.env` 或设置系统环境变量，如 `PORT`、`HOST`、`NODE_ENV` 等。

4. **启动服务**

   - 直接运行：`npm start`
   - 或使用 PM2：`pm2 start src/index.js --name carpooling-api`

5. **若使用 Nginx 反向代理**  
   将对外域名（如 `https://api-test.xxx.com`）反向代理到 `http://127.0.0.1:3000`。

---

## 四、流程小结

| 步骤 | 在哪里 | 做什么 |
|------|--------|--------|
| 1 | 本机 backend | `npm run build` 得到 dist/ |
| 2 | 本机 → 腾讯云服务器（由陈柯上传） | 在服务器上打开部署目录，将 dist 内容通过文件传输上传 |
| 3 | 服务器部署目录 | `npm install --production` |
| 4 | 服务器 | 配置环境变量后 `npm start` 或 `pm2 start ...` |

更细的说明（环境变量、PM2、反向代理）见 [backend/README.md](../backend/README.md)。
