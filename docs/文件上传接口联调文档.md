# 文件上传接口联调文档

## 1. 文档说明

- 适用范围：前端联调文件上传功能（用户头像、车辆照片等）
- 服务端实现：Node.js + Express + Multer
- 响应结构：统一返回 `code / message / data / requestId`

## 2. 环境与基础地址

- 开发环境（本地）：`http://localhost:3000`
- 测试环境（公网服务器）：`http://你的公网IP:3000`

前端应基于环境变量（如 `EXPO_PUBLIC_API_URL`）拼接完整请求地址。

## 3. 接口总览

1. 上传文件：`POST /api/upload`
2. 访问文件：`GET /uploads/{filename}`

---

## 4. 上传文件

### 4.1 请求信息

- 方法：`POST`
- 路径：`/api/upload`
- `Content-Type`：`multipart/form-data`

### 4.2 请求参数

#### 必填参数

- `file`：文件字段（FormData），支持单文件上传

#### 可选参数

- `x-request-id`：请求头，全链路追踪ID（前端生成，后端兜底）

### 4.3 文件限制

- 支持格式：JPG、PNG、WEBP
- 文件大小：最大 5MB
- 单次上传：仅支持单文件

### 4.4 请求示例

#### 前端示例（React Native）

```javascript
const uploadFile = async (fileUri) => {
  const formData = new FormData();
  formData.append('file', {
    uri: fileUri,
    type: 'image/jpeg',
    name: 'avatar.jpg',
  });

  const response = await fetch('http://localhost:3000/api/upload', {
    method: 'POST',
    headers: {
      'x-request-id': generateRequestId(), // 可选
    },
    body: formData,
  });

  return await response.json();
};
```

#### cURL 示例

```bash
curl -X POST http://localhost:3000/api/upload \
  -H "x-request-id: RN202603211430001" \
  -F "file=@/path/to/image.jpg"
```

### 4.5 响应示例

#### 成功响应（200）

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "url": "http://localhost:3000/uploads/1711234567890-a1b2c3d4e5f6g7h8.jpg"
  },
  "requestId": "RN202603211430001"
}
```

#### 失败响应（400 - 文件类型不支持）

```json
{
  "code": 400,
  "message": "仅支持 JPG、PNG、WEBP 格式",
  "data": null,
  "requestId": "RN202603211430001"
}
```

#### 失败响应（400 - 文件过大）

```json
{
  "code": 400,
  "message": "文件大小不能超过 5MB",
  "data": null,
  "requestId": "RN202603211430001"
}
```

#### 失败响应（400 - 未接收到文件）

```json
{
  "code": 400,
  "message": "未接收到文件",
  "data": null,
  "requestId": "RN202603211430001"
}
```

#### 失败响应（500 - 服务异常）

```json
{
  "code": 500,
  "message": "上传失败，请稍后重试",
  "data": null,
  "requestId": "BE202603211430001"
}
```

## 5. 访问文件

### 5.1 请求信息

- 方法：`GET`
- 路径：`/uploads/{filename}`
- 说明：直接访问静态资源，无需认证

### 5.2 请求示例

```
http://localhost:3000/uploads/1711234567890-a1b2c3d4e5f6g7h8.jpg
```

浏览器或 `<Image>` 组件直接使用该 URL 即可显示图片。

---

## 6. 业务说明

### 6.1 文件存储

- 文件保存在服务器 `backend/uploads/` 目录
- 文件名格式：`时间戳-随机字符串.扩展名`（防止冲突）
- 文件通过静态文件服务对外暴露

### 6.2 数据库关联（待实现）

当前版本仅实现文件上传与访问，URL 与用户的关联（如头像字段）需等数据库模块完成后补充：

- 预留位置：`backend/src/service/upload-service.js` 的 `handleUpload` 函数
- 待补充逻辑：调用 dao 层将 `url` 写入用户表

### 6.3 安全建议

- 生产环境建议配置 Nginx 反向代理，限制上传速率
- 建议增加用户认证，防止匿名上传
- 建议定期清理未关联的文件

## 7. 环境变量配置

在 `backend/.env` 中配置：

```env
# 服务器基础URL（用于生成文件访问地址）
SERVER_BASE_URL=http://你的公网IP:3000

# 服务端口
PORT=3000
```

## 8. 测试步骤

1. 启动后端服务：`cd backend && npm run dev`
2. 使用 Postman 或 cURL 上传测试图片
3. 复制返回的 `url`，在浏览器中访问验证

## 9. 常见问题

**Q: 上传后无法访问图片？**
A: 检查 `SERVER_BASE_URL` 是否配置正确，确保 `/uploads` 路径可访问

**Q: 文件上传失败，提示 500？**
A: 检查 `backend/uploads/` 目录是否存在且有写入权限

**Q: 如何关联到用户头像？**
A: 等数据库模块完成后，在 `upload-service.js` 的 TODO 位置补充 dao 调用
