# 短信验证接口联调文档（前端）

## 1. 文档说明

- 适用范围：前端联调短信验证码发送与校验
- 服务端实现：Node.js + Express，接口统一为 `POST`
- 响应结构：统一返回 `code / message / data / requestId`

## 2. 环境与基础地址

- 开发环境（本地）：`http://localhost:3000`
- 测试环境（示例）：`https://api-test.yourdomain.com`

前端应基于环境变量（如 `EXPO_PUBLIC_API_URL`）拼接完整请求地址。

## 3. 接口总览

1. 发送验证码：`POST /api/sms/send`
2. 校验验证码：`POST /api/sms/verify`

---

## 4. 发送验证码

### 4.1 请求信息

- 方法：`POST`
- 路径：`/api/sms/send`
- `Content-Type`：`application/json`

### 4.2 请求参数

#### 必填参数

- `phoneNumber`：手机号（字符串）
- `signName`：短信签名（字符串）
- `templateCode`：短信模板编码（字符串）
- `templateParam`：模板变量 JSON 字符串（字符串）

#### 可选参数

- `schemeName`：验证码服务名称（字符串）
- `validTime`：验证码有效期（秒）
- `interval`：发送间隔（秒）
- `codeLength`：验证码长度
- `codeType`：验证码类型
- `duplicatePolicy`：重复验证码策略
- `outId`：外部流水号
- `returnVerifyCode`：是否返回验证码（布尔值，联调用建议 `true`）
- `smsUpExtendCode`：上行扩展码

### 4.3 请求示例

```json
{
  "phoneNumber": "18975949538",
  "signName": "云渚科技验证平台",
  "templateCode": "100001",
  "templateParam": "{\"code\":\"##code##\",\"min\":\"5\"}",
  "returnVerifyCode": true
}
```

> 注意：`templateParam` 是“字符串”，不是对象。  
> 示例模板：“您的验证码是${code}，有效期${min}分钟，请勿告诉他人。”

### 4.4 成功响应示例

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "success": true,
    "aliyunCode": "OK",
    "aliyunMessage": "OK",
    "verifyCode": "5973"
  },
  "requestId": "RN-277bd8cc-8f72-44ec-8663-7d6fb66fdcd6"
}
```

### 4.5 失败响应示例

```json
{
  "code": 400,
  "message": "缺少必要参数：phoneNumber、signName、templateCode、templateParam",
  "data": null,
  "requestId": "RN-xxxx"
}
```

```json
{
  "code": 500,
  "message": "短信发送失败",
  "data": {
    "success": false,
    "aliyunCode": "xxx",
    "aliyunMessage": "xxx",
    "requestId": "阿里云请求ID"
  },
  "requestId": "RN-xxxx"
}
```

## 5. 校验验证码

### 5.1 请求信息

- 方法：`POST`
- 路径：`/api/sms/verify`
- `Content-Type`：`application/json`

### 5.2 请求参数

#### 必填参数

- `phoneNumber`：手机号（字符串）
- `verifyCode`：验证码（字符串）

#### 可选参数

- `schemeName`：验证码服务名称（字符串）
- `caseAuthPolicy`：大小写校验策略
- `outId`：外部流水号

### 5.3 请求示例

```json
{
  "phoneNumber": "18975949538",
  "verifyCode": "5973"
}
```

### 5.4 成功响应示例

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "success": true,
    "aliyunCode": "OK",
    "aliyunMessage": "OK",
    "verifyResult": "PASS",
    "isValid": true,
    "tempToken": "eyJhbGciOiJIUzI1NiIs..."
  },
  "requestId": "RN-03ac6891-0b1e-4173-be54-c60b3b5da53a"
}
```

### 5.5 失败响应示例

```json
{
  "code": 400,
  "message": "缺少必要参数：phoneNumber、verifyCode",
  "data": null,
  "requestId": "RN-xxxx"
}
```

```json
{
  "code": 500,
  "message": "短信校验失败",
  "data": {
    "success": false,
    "aliyunCode": "xxx",
    "aliyunMessage": "xxx",
    "verifyResult": "UNKNOWN",
    "requestId": "阿里云请求ID"
  },
  "requestId": "RN-xxxx"
}
```

## 6. 前端联调建议

1. 先调发送接口，确认 `code=200` 且 `data.success=true`
2. 联调阶段可开启 `returnVerifyCode=true`，直接使用返回的 `verifyCode` 调校验接口
3. 注册场景：校验成功响应含 `isValid`、`tempToken`，下一步提交 `POST /api/auth/register`（见 [用户注册接口联调文档.md](./用户注册接口联调文档.md)）
3. 校验通过以 `data.verifyResult === "PASS"` 为准
4. 每次请求记录 `requestId`，便于后端排查问题

## 7. 常见问题

- `无法连接到远程服务器`：后端未启动或基础地址错误
- `code=500 且凭据相关报错`：后端未正确配置阿里云 AK/SK
- `code=500 且 aliyunMessage 非 OK`：签名、模板、参数或服务方案配置不匹配
