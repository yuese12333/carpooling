# 系统架构设计文档

## 一、系统概述

### 1.1 系统目标
Carpooling 拼车平台是一个面向校园用户的拼车服务应用，提供行程发布、搜索匹配、在线预订、支付结算等核心功能。

### 1.2 设计原则
- **前后端分离**：前端专注 UI 交互，后端专注业务逻辑
- **模块化设计**：功能模块独立，低耦合高内聚
- **可扩展性**：支持功能扩展和性能扩展
- **安全性**：数据加密、权限控制、操作审计

## 六、安全架构

### 6.1 认证流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  用户    │────▶│  登录    │────▶│ JWT签发  │────▶│ 返回Token│
└──────────┘     └──────────┘     └──────────┘     └──────────┘
                                                        │
                                                        ▼
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  业务响应│◀────│  业务处理│◀────│  鉴权通过│◀────│ 携带Token│
└──────────┘     └──────────┘     └──────────┘     └──────────┘
```

### 6.2 安全措施

| 安全层面 | 措施 |
|----------|------|
| 传输安全 | HTTPS 加密传输 |
| 认证安全 | JWT Token + Refresh Token |
| 密码安全 | bcrypt 哈希存储 |
| 数据安全 | 敏感信息脱敏 |
| 日志安全 | requestId 链路追踪 |
| 权限安全 | RBAC 角色权限控制 |
