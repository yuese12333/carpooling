# 拼车平台项目计划书

## 一、项目概述

### 1.1 项目名称
Carpooling - 校园拼车平台

### 1.2 项目背景
随着共享经济的发展，拼车出行已成为一种经济、环保的出行方式。本项目旨在开发一款移动端拼车应用，为用户提供便捷的行程发布、搜索、匹配服务。

### 1.3 项目目标
- 实现用户注册、登录、个人信息管理功能
- 实现行程发布、搜索、预订、评价完整业务闭环
- 实现管理后台对用户和内容的管理
- 支持高德地图定位与路线规划
- 实现前后端分离架构，支持 Mock 模式独立开发

## 三、技术方案

### 3.1 技术栈选型

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| 前端框架 | React Native + Expo | 跨平台开发，一套代码运行 iOS/Android |
| 前端路由 | Expo Router | 文件系统路由，开发体验好 |
| 前端状态 | Zustand | 轻量级状态管理，学习成本低 |
| 后端框架 | Node.js + Express | JavaScript 全栈，团队熟悉 |
| 数据库 | MySQL | 关系型数据库，事务支持完善 |
| ORM | Prisma | 类型安全，迁移管理方便 |
| 认证 | JWT | 无状态认证，适合移动端 |
| 地图服务 | 高德地图 SDK | 国内定位准确，API 完善 |
