## 一、测试环境

### 1.1 环境配置

| 环境 | 说明 | 配置 |
|------|------|------|
| 开发环境 | 本地开发测试 | `NODE_ENV=development` |
| 测试环境 | 集成测试 | `NODE_ENV=test` |
| Mock 模式 | 前端无后端测试 | `isMockMode=true` |
| API 模式 | 前后端联调 | `isMockMode=false` |

### 1.2 测试工具

| 类型 | 工具 | 用途 |
|------|------|------|
| 后端单元测试 | Jest | Service/DAO 层测试 |
| 后端集成测试 | Supertest + Jest | API 接口测试 |
| 前端组件测试 | Jest + React Testing Library | 组件渲染测试 |
| E2E 测试 | Detox / Maestro | 端到端流程测试 |
| 接口测试 | Postman / cURL | 手动接口验证 |
| 性能测试 | Artillery / k6 | 负载与压力测试 |

### 1.3 测试数据库

```bash
# 创建测试数据库
CREATE DATABASE carpooling_test DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 环境变量
DATABASE_URL=mysql://root:password@localhost:3306/carpooling_test

## 六、E2E 端到端测试

### 6.1 测试流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   注册流程   │ → │   登录流程   │ → │   业务流程   │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 6.2 核心业务流程测试

#### 6.2.1 注册→登录流程

```
1. 打开 App
2. 点击"注册"
3. 输入昵称 → 校验可用性
4. 输入手机号 → 获取验证码
5. 输入验证码 → 验证通过
6. 设置密码 → 提交注册
7. 自动登录 → 跳转首页
8. 退出登录
9. 使用手机号+密码登录
10. 验证跳转首页
```

#### 6.2.2 搜索→预订流程

```
1. 登录成功
2. 点击"找拼车"
3. 输入起点、终点
4. 选择日期
5. 点击搜索
6. 验证结果列表
7. 点击行程卡片
8. 验证详情页信息
