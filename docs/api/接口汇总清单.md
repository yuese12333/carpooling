# 接口汇总清单

> 版本：v1.0

---

## 一、路径命名约定

| 规则 | 说明 | 示例 |
|------|------|------|
| 统一前缀 | 所有接口使用 `/api/` | `/api/users/me` |
| 资源复数 | 集合资源用复数名词 | `/api/rides`、`/api/vehicles` |
| 动词禁止出现在路径中 | 用 HTTP 方法表达操作 | `POST /api/rides` |
| 子资源用斜杠 | 从属关系用路径嵌套 | `/api/vehicles/:id/default` |
| 操作例外 | 非 CRUD 动作可用子路径动词 | `/api/auth/logout`、`/api/sms/send` |
| 连字符分隔 | 多词路径用 `-` 不用 `_` | `/api/find-ride` |

## 三、状态图例

| 标记 | 含义 |

| ⏳ 待联调 | 后端已实现，待联调验证 |

## 四、推荐联调顺序

```
阶段一（登录，✅ 已完成前后端适配）
  GET  /api/auth/login/config
  POST /api/auth/login/password
  详见 api/登录接口联调文档.md

阶段二 A（忘记密码）
  POST /api/auth/check-phone
  POST /api/auth/password/sms
  POST /api/auth/password/verify-code   ← 后端待实现
  POST /api/auth/password/reset         ← 后端待实现

阶段二 B（文件上传，随时可插空）
  POST /api/upload

阶段三（管理后台，✅ 已完成联调）
  GET  /api/admin/users
  POST /api/admin/users/status
  POST /api/admin/users/role

阶段四（核心业务，按 App 打开后的自然路径，后端待实现）
  首页 /api/home
    → GET /api/home/user-info
    → GET /api/home/statistics
  通知 /api/notifications
    → GET /api/notifications/unread-status

  我的行程 /api/trips
    → GET /api/trips

    → POST /api/trips/:id/rate
```
