# 登录接口联调文档

## 1. 文档说明

- **适用范围**：App 登录页（账号密码 + 动态配置）
- **阶段**：接口汇总清单「阶段一」
- **服务端**：`auth-router` → `auth-controller` → `auth-service`
- **响应结构**：统一 `code` / `message` / `data` / `requestId`（HTTP 200 时 `code === 200` 表示成功）

## 2. 环境与基础地址

| 环境 | 配置 |
|------|------|
| 前端 | `frontend/.env` → `EXPO_PUBLIC_API_URL=http://1.15.45.125`（公网示例） |
| 后端 | `backend/.env` → `PORT`、`DATABASE_URL`、JWT 密钥等 |

联调前在登录页将 **Mock 开关切到「正式接口」**（`isMockMode = false`），并重启 Expo 使 `.env` 生效。

所有请求建议携带 Header：`X-Request-Id: <链路 ID>`。

## 3. 接口清单

| 步骤 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 1 | GET | `/api/auth/login/config` | 登录页标题、副标题、第三方入口 |
| 2 | POST | `/api/auth/login/password` | 手机号 + 密码登录 |

## 4. 接口明细

### 4.1 登录页配置

- **GET** `/api/auth/login/config`
- Query（可选）：`appVersion`、`platform`（`ios` / `android` / `web`）

成功 `data` 示例：

```json
{
  "logoUrl": "https://example.com/logo.png",
  "title": "Carpooling",
  "subtitle": "安全便捷拼车",
  "activeSocialPlatforms": ["wechat", "qq"]
}
```

### 4.2 密码登录

- **POST** `/api/auth/login/password`

请求体：

```json
{
  "phone": "13812345678",
  "password": "abc12345",
  "rememberMe": false
}
```

| 字段 | 说明 |
|------|------|
| phone | 11 位手机号 |
| password | 非空，最长见后端 `MAX_PASSWORD_LENGTH` |
| rememberMe | 可选布尔；`true` 时 access/refresh 有效期更长 |

成功 `data`：

```json
{
  "token": "<JWT>",
  "refreshToken": "<JWT>",
  "userId": "u_xxxxxxxx",
  "userName": "昵称",
  "avatarUrl": "",
  "expireIn": 604800
}
```

常见错误：

| HTTP | code | 说明 |
|------|------|------|
| 400 | 400 | 参数校验失败（手机号、密码、rememberMe 类型） |
| 401 | 401 | 用户不存在 / 密码错误 / 账号已禁用 |
| 500 | 500 | 服务异常 |

## 5. 前端对接参考

| 模块 | 路径 |
|------|------|
| API | `frontend/src/api/auth.ts`（`fetchLoginConfig`、`loginByPassword`） |
| 页面 | `frontend/src/pages/auth/login/login.tsx` |
| Hook | `frontend/src/hooks/use-login-form.ts` |
| 鉴权态 | `frontend/src/store/auth-context.tsx` |

**响应适配（已联调修正）**：

- 后端无 `success` 字段，前端用 `isApiSuccess`（`code === 200`）判断。
- `loginByPassword` 成功时解包 `data`，写入 `token` / `refreshToken` / `userId` / `userName`。
- JWT payload 含 `role`；管理员登录后跳转 `/admin/users`。
